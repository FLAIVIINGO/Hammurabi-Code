Binary Search

Andrew Babilonia

In this blog post, we are going to take a look at the binary search algorithm. There are some prerequisites here that must be met in order to work. The array being searched must be sorted, and in particular for this algorithm, it will be sorted in ascending order. This algorithm uses a two pointer system. We have a left pointer that starts at 0 and the right pointer will start at the array's length minus one to account for the fact that we count zero as an index. The idea behind binary search is to start from the middle of the array, compare the target value and the value at that array's position, then make a decision based on that comparison. If we find that the target value matches, we return that the array holds the target value. If this condition fails, there are two routes that we could take based on the comparison of the target value and the array value. If the target is less than the middle indexed element, we can safely assume that the target value cannot exist any higher than our middle indexed element. Therefore, we can move our highest indexed element, which is named right, to one index lower. The same can be said about the last condition, which would be met if the target value is greater than the middle indexed value. If this condition is met, we can assume that the value cannot exist below the middle indexed value, so we move our left index up by one. We repeat these steps until either the target is found, which would return the index of the element, or we would exhaust our search. This means that the left and right value equal each other. If this happens, our array does not contain the value in question, and we can return a minus one. Below is the code.

public int binarySearch(int[] numbers) {
    int left = 0;
    int right = numbers.lenght;

    while(left <= right) {
        int mid = left + ((right - left) / 2);
        int midElement = numbers[mid];
        if(target > midElement)
            left = mid + 1;
        else if(target < midElement)
            right = mid - 1;
        else
            return mid;
    }
    return -1;
}

In conclusion, the worst case time complexity of binary search would be O(logN). With each iteration, our array size halves in available indices. This is a useful algorithm to know, as it decreases our search time from O(N) to logN.
